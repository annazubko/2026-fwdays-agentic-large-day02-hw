# Skills: ручний vs Repomix

## Структура SKILL.md

| Аспект | Ручний (`memory-bank-update`, `build-verify`) | Repomix (`excalidraw-ref`) |
|--------|-----------------------------------------------|---------------------------|
| `name` у frontmatter | Коротке: `memory-bank-update` | Авто-генероване довге: `repomix-reference-2026-...` |
| `description` | Написана людиною, точно описує тригери | Шаблонна, загальна |
| Секції | `When to use`, `Inputs`, `Steps`, `Outputs`, `Safety` | `Overview`, `Files`, `How to Use`, `Tips` |
| Кроки виконання | Конкретні, з командами | Інструкції де шукати в reference-файлах |
| Safety rules | Є явні обмеження | Відсутні |

## Підхід до знань

**Ручний skill** — процедура: агент знає *що робити* крок за кроком.

**Repomix skill** — довідник: агент знає *де знайти* інформацію через `references/files.md`, `project-structure.md` тощо.

```
Ручний:   Агент → виконує кроки → результат
Repomix:  Агент → читає reference-файли → розуміє кодову базу
```

## Розмір і токени

| | Ручний | Repomix |
|-|--------|---------|
| `SKILL.md` | ~40 рядків | ~80 рядків |
| Додаткові файли | Немає | 4 файли (`summary.md`, `project-structure.md`, `files.md`, `tech-stack.md`) |
| Загальний обсяг | Мінімальний | ~3,4M токенів у `files.md` |

## Висновок

Для Claude Code з доступом до файлової системи підхід Repomix надлишковий — агент і так читає файли напряму. Repomix-skills корисні для агентів **без** інструментів читання файлів.

Ручні skills кращі для автоматизації конкретних процесів з чіткими кроками, обмеженнями і очікуваним результатом.
